# 구현 작업 목록 (Task List)

- [x] **Phase 1: 프로젝트 셋업 및 타로 DB/스키마 구축**
  - [x] 프로젝트 뼈대(웹앱 프레임워크 또는 Vanilla HTML/CSS/JS)를 생성하고 초기화합니다.
  - [x] `/resource` 내의 CSV 파일과 `/image` 내의 22장 메이저 아르카나 이미지만을 연결하는 JSON 형식의 정적 데이터를 구축합니다.
  - [x] 전역적으로 사용할 따뜻한 감성의 CSS 색상 팔레트 및 디자인 시스템(폰트, 변수 등)을 설정합니다.

- [x] **Phase 2: 카드 1장 추출 핵심 로직 및 UI 완료**
  - [x] 버튼 클릭 시 22장의 데이터 중 `random.choice` 방식의 난수로 1장을 뽑는 핵심 상태 관리 로직을 개발합니다.
  - [x] 감성적인 메인 화면 및 부드러운 전환 효과(카드 뒤집기 애니메이션)를 부여합니다.
  - [x] 결과 페이지에서 공감-통찰-확언 3단계 텍스트가 순차적으로 시각적으로 돋보이게 렌더링되도록 디자인을 다듬습니다.

- [x] **Phase 3: 9:16 인스타그램 공유용 뷰(바이럴 엔진) 개발**
  - [x] 화면 상에 9:16 인스타그램 스토리 비율의 숨겨지거나 노출되는 '캡쳐용 뷰어 컴포넌트(카드 이미지 + 한 줄 확언 + 링크 워터마크)'를 조판합니다.
  - [x] `html2canvas` 또는 브라우저 네이티브 API를 활용하여, 해당 뷰어를 이미지로 저장 및 모바일 공유를 유도하는 기능을 완성합니다.

- [x] **Phase 4: 유료 플랜용 컨시어지(Kakao/Email) 외부 링크 연결 및 마감**
  - [x] 결과 확인 UI 최하단에 '전문가에게 심층 배열법 요청하기(유료)' Call To Action (CTA) 버튼을 배치합니다.
  - [x] 사용자가 미리 지정된 카카오톡 오픈채널 링크 또는 Email 작성창(`mailto:`)으로 빠져나가는 트리거를 붙이고, 정상 동작 여부를 검증합니다.
  - [x] 전반적인 반응형 테스트 및 최종 빌드 확인을 진행합니다.
