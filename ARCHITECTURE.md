# 시스템 및 UI/UX 설계도 (Architecture & UI Design)

## 1. 데이터베이스 스키마 및 핵심 로직 설계
- **형태:** 서버/DB 구축 리소스를 최소화하기 위해 앱 번들에 포함된 정적 JSON 배열(또는 JS/TS 객체) 사용.
- **데이터 구조 (1909 Rider-Waite 메이저 아르카나 22장):**
  ```json
  [
    {
      "id": 0,
      "name": "0. 바보 (The Fool)",
      "image_url": "/image/0. 바보 (The Fool).jpg",
      "step1_empathy": "정해진 길이 없어 막막하고 조금은 두려운 기분이 드시나요?",
      "step2_insight": "이 카드는 낭떠러지 앞에서도 두려움 없는 순수한 시작과 무한한 잠재력을 의미합니다.",
      "step3_affirmation": "오늘은 타인의 시선이나 낡은 규칙은 내려놓고, 당신의 직관이 이끄는 대로 가벼운 첫발을 내디뎌보세요."
    }
  ]
  ```
- **난수 무작위 추출 로직:**
  플랫폼/언어에 내장된 난수 생성기 활용. 프론트엔드(React/Vanilla JS 등)에서 배열 길이만큼 랜덤 인덱스 1개를 생성.
  *예시:* `const randomCard = tarotData[Math.floor(Math.random() * tarotData.length)];`

## 2. UI/UX 컨셉 및 바이럴 성장 엔진 (Viral Engine) 설계
- **UI 디자인 언어 (Aesthetics):**
  - 기존의 딱딱하고 어두운 오컬트풍 타로를 완전히 탈피.
  - 현대적이고 감성적인 색상 팔레트 (따뜻한 파스텔, 오로라 빛의 부드러운 그라데이션) 및 글래스모피즘(Glassmorphism) 적극 사용.
  - 사용자가 앱에 접속하는 순간부터 '정서적 연결감'과 '위로'를 느낄 수 있는 미세한 마이크로 애니메이션 제공.
- **바이럴 레이아웃 영역 (Instagram Story 9:16 최적화):**
  - **비율:** `9:16` 모바일 화면 비율 고정.
  - **구조:** 화면 캡처 라이브러리(html2canvas 등)로 렌더링될 타겟 노드(DOM).
    - *Header:* '오늘의 수호 카드' 등 심플하고 고풍스러운 헤더
    - *Center:* 아름답게 프레임 처리된 카드 이미지 (그림자 효과 적용)
    - *Bottom:* 3단계 문구 중 가장 긍정적인 '확언(Affirmation)' 문구 크게 강조 (아름다운 폰트 사용)
    - *Footer:* 웹앱 유입을 위한 로고 워터마크 및 짧은 링크 URL
  - 버튼 클릭 1회만으로 위 레이아웃 영역만 깨끗하게 캡처 및 이미지 다운로드가 이루어져, 즉각적인 스토리에 업로드가 가능하도록 설계.
